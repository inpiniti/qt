# 코드 리뷰 워크플로우

이 워크플로우는 코드 변경 후 품질을 검증하는 단계입니다.

## 리뷰 대상

- 새로 작성된 코드
- 수정된 기존 코드
- 추가된 의존성

## 리뷰 체크리스트

### 1. React 규칙 준수
- [ ] 컴포넌트가 순수 함수인가? (사이드 이펙트 분리)
- [ ] Props와 State가 불변으로 처리되는가?
- [ ] Hook이 최상위 레벨에서만 호출되는가?
- [ ] 메모이제이션이 적절히 사용되었는가? (useMemo, useCallback)

### 2. UI/UX 규칙 준수
- [ ] shadcn/ui 컴포넌트 사용 중인가?
- [ ] Native Dialog(alert, confirm) 대신 shadcn Dialog 사용?
- [ ] Tailwind CSS 스타일링 일관성 유지?
- [ ] 다크 테마 호환성 확인?

### 3. API 호출 규칙 준수
- [ ] 외부 API가 프록시를 통해 호출되는가?
- [ ] CORS 에러 발생 가능성 없는가?
- [ ] 에러 핸들링이 적절한가?
- [ ] 캐싱 전략이 적용되었는가?

### 4. 상태 관리 규칙 준수
- [ ] Zustand 스토어 패턴을 따르는가?
- [ ] 상태 직접 변경 없이 set 함수 사용?
- [ ] 불필요한 리렌더링 방지 조치?

### 5. 성능 최적화
- [ ] 무한 루프 가능성 없는가?
- [ ] 대용량 데이터 처리 시 최적화 적용?
- [ ] React DevTools로 렌더링 병목 확인?
