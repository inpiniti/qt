---
description: 변경된 코드 스테이징, 커밋 및 원격 저장소 푸시 워크플로우
---

이 워크플로우는 작업한 내용을 안전하게 Git에 커밋하고 푸시하는 과정을 안내합니다.

1. **상태 확인**: 현재 변경된 파일 목록을 확인합니다.
// turbo
2. **스테이징**: 모든 변경 사항을 스테레징 영역에 추가합니다.
   - `git add .`
3. **커밋 메시지 생성**: `GEMINI.md`에 명시된 Udacity + Gitmoji 컨벤션을 준수하며, **한국어**로 작성합니다.
   - 구조: `<Emoji> <Type>: <Subject>`
   - 예: `✨ Feat: 시뮬레이션 엔진에 V-Martingale 전략 추가`
   - 본문: 구체적으로 무엇이, 왜 변경되었는지 한국어로 작성
// turbo
4. **커밋 실행**: 작성된 메시지로 커밋을 진행합니다.
   - `git commit -m "[Emoji] [Type]: [Subject]" -m "[Body]"`
// turbo
5. **원격 저장소 푸시**: 현재 브랜치 내용을 원격 서버로 전송합니다.
   - `git push origin [현재브랜치명]`

---
**주의**: 푸시 전 반드시 `npm run build` 등을 통해 빌드 오류가 없는지 확인하는 것을 권장합니다.
