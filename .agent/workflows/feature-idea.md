# 💡 기능 아이디어 제안 워크플로우

이 워크플로우를 실행하면 에이전트가 프로젝트를 분석하여 투자자에게 도움이 될 수 있는 새로운 기능 아이디어를 제안합니다.

## 사용 시점

- 프로젝트에 새 기능을 추가하고 싶을 때
- 투자 도구로서 어떤 기능이 부족한지 파악하고 싶을 때
- 주기적인 기능 로드맵 점검 시

## 분석 단계

### 1. 현재 프로젝트 상태 파악

**필수 확인 파일:**
- `README.md` - 현재 구현된 기능 목록
- `src/components/` - UI 컴포넌트 구조
- `src/lib/` - 핵심 로직 및 API

### 2. 분석 관점 (투자자 페르소나)

다음 투자자 유형별로 필요한 기능을 검토합니다:

```markdown
## 투자자 페르소나

### 🔰 초보 투자자
- 투자 용어 설명이 충분한가?
- 리스크 경고가 명확한가?
- 학습 자료/가이드가 있는가?

### 📊 기술적 분석가
- 보조지표가 충분한가? (MACD, Stochastic 등)
- 차트 도구가 다양한가?
- 백테스팅이 상세한가?

### 📰 펀더멘털 분석가
- 재무제표 정보가 있는가?
- 실적 발표 캘린더가 있는가?
- 산업 비교 분석이 가능한가?

### 🤖 자동화 투자자
- 자동 매매 전략이 다양한가?
- 알림/푸시가 있는가?
- 포트폴리오 리밸런싱 기능이 있는가?

### 💰 리스크 관리자
- 손익 추적이 용이한가?
- 분산투자 분석이 있는가?
- 최대 낙폭(MDD) 계산이 있는가?
```

### 3. 기능 Gap 분석

현재 프로젝트에서 **있는 기능**과 **없는 기능**을 비교합니다:

```markdown
## Gap 분석 템플릿

### ✅ 현재 있는 기능
- [ ] 가격 차트 표시
- [ ] AI 가격 예측
- [ ] 뉴스 감성 분석
- [ ] 매매 시뮬레이션
- [ ] 자동 매매
- [ ] ...

### ❌ 현재 없는 기능
- [ ] ???
- [ ] ???
```

### 4. 기능 제안 리포트 작성

분석 결과를 다음 형식으로 리포팅합니다:

```markdown
## 🎯 기능 제안 리포트

### 제안 1: [기능명]

**현재 상황 (As-Is)**
[현재 어떤 기능이 없거나 부족한지]

**제안 기능 (To-Be)**
[어떤 기능을 추가하면 좋을지]

**기대 효과**
[투자자에게 어떤 도움이 되는지]

**구현 난이도**: ⭐⭐⭐ (1~5)

**우선순위**: 높음 / 중간 / 낮음

---

### 제안 2: [기능명]
...
```

## 분석 시 참고할 경쟁 서비스

- TradingView: 차트/지표
- Yahoo Finance: 뉴스/재무 정보
- Seeking Alpha: 분석 리포트
- Robinhood: 간편한 UX
- 토스증권: 한국 투자자 친화 UI

## 기능 카테고리별 체크리스트

### 📈 차트/분석
- [ ] 캔들스틱 차트 (현재: 라인 차트만)
- [ ] 추가 보조지표 (MACD, Stochastic, ADX 등)
- [ ] 다중 시간대 비교
- [ ] 드로잉 도구 (추세선, 피보나치 등)

### 📰 정보/뉴스
- [ ] 실적 발표 캘린더
- [ ] 배당 정보
- [ ] 재무제표 (매출, 순이익 등)
- [ ] SEC 공시 (10-K, 10-Q)

### 🔔 알림
- [ ] 가격 도달 알림
- [ ] 매매 신호 알림
- [ ] 뉴스 속보 알림
- [ ] 포트폴리오 변동 알림

### 💼 포트폴리오
- [ ] 전체 자산 현황 대시보드
- [ ] 수익률 그래프 (일별/월별)
- [ ] 섹터별 분산 분석
- [ ] 리밸런싱 제안

### 🛡️ 리스크 관리
- [ ] 최대 낙폭(MDD) 계산
- [ ] 변동성(VaR) 분석
- [ ] 상관관계 분석
- [ ] 손절/익절 자동화

### 📚 교육/학습
- [ ] 투자 용어 사전
- [ ] 전략별 튜토리얼
- [ ] 모의 투자 모드
- [ ] 과거 사례 분석

## 리포트 출력 형식

분석 완료 후 다음 형식으로 사용자에게 리포팅합니다:

```markdown
# 📋 기능 아이디어 분석 리포트

## 분석 일시
YYYY-MM-DD HH:mm

## 분석 요약
- 현재 기능 수: N개
- 제안 기능 수: N개
- 최우선 추천: [기능명]

## 상세 제안

### 🥇 우선순위 1: [기능명]
- 현황: ...
- 제안: ...
- 기대효과: ...
- 난이도: ⭐⭐

### 🥈 우선순위 2: [기능명]
...

### 🥉 우선순위 3: [기능명]
...

## 구현 로드맵 제안
1. 단기 (1주): ...
2. 중기 (1개월): ...
3. 장기 (3개월): ...
```

## 주의사항

- 코드 작성 없이 **분석 및 리포팅만** 수행
- 구현 가능성과 ROI를 고려하여 제안
- 기존 프로젝트 아키텍처와의 적합성 검토
- 사용자가 원하면 `/feature-dev` 워크플로우로 연계
