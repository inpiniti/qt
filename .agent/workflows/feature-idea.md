# 💡 맞춤형 기능 아이디어 제안 워크플로우

이 워크플로우를 실행하면 에이전트가 현재 프로젝트의 성격, 기술 스택, UI 구조를 분석하여 프로젝트의 가치를 높일 수 있는 새로운 기능 아이디어를 제안합니다.

## 사용 시점

- 프로젝트에 어떤 기능을 추가할지 막막할 때
- 현재 개발 중인 서비스의 완성도를 높이고 싶을 때
- 프로젝트의 목적에 맞는 사용자 경험(UX) 개선이 필요할 때

## 분석 단계

### 1. 현재 프로젝트 문맥 파악

**필수 확인 파일:**
- `README.md` - 프로젝트의 전체 목적 및 현재 기능
- `package.json` - 사용 중인 기술 스택 및 외부 라이브러리
- `src/components/layout/` - 현재 구축된 레이아웃 및 내비게이션 구조
- `src/store/` 또는 `src/hooks/` - 데이터 흐름 및 상태 관리 패턴

### 2. 다각도 분석 (Multi-View Analysis)

프로젝트를 다음 세 가지 관점에서 분석합니다.

```markdown
## 분석 관점

### 🎯 서비스 본질 (Core Value)
- 이 서비스가 해결하려는 핵심 문제는 무엇인가?
- 사용자가 가장 자주 머무는 화면은 어디인가?
- 핵심 흐름(Core Flow)에서 이탈이 발생할 만한 구간은?

### 🛠 기술적 확장성 (Technical Leap)
- 현재 스택(Next.js, Tailwind, Zustand 등)으로 쉽게 구현 가능한 고부가가치 기능은?
- 아직 활용하지 않은 최신 라이브러리나 API 연동 가능성은? (AI, 데이터 시각화 등)

### 🎨 디자인 및 사용성 (Design & UX)
- 현재 레이아웃에서 정보 밀도가 너무 낮거나 높은 곳은?
- 마이크로 인터랙션이나 애니메이션으로 사용성을 높일 구간은?
- 프로젝트의 '엔지니어링 느낌'을 더해줄 디테일은?
```

### 3. 기능 제안 리포트 작성

현재 프로젝트에 **가장 잘 어울리는** 기능을 다음 형식으로 제안합니다.

```markdown
## 🎯 맞춤형 기능 제안 리포트

### 제안 1: [기능명]

**제안 배경 (Context)**
[현재 프로젝트의 어떤 부분과 시너지가 날 수 있는지]

**상세 제안 (What to Build)**
[구현할 기능의 구체적인 모습과 동작 방식]

**프로젝트 가치 향상 (Value)**
[이 기능이 추가되었을 때 사용자가 얻는 이점]

**기술적 검토 (Technical Detail)**
[사용 중인 기술 스택 내에서의 구현 전략]

**우선순위**: 상 / 중 / 하
```

### 4. 구현 로드맵 제안

제안된 기능들을 단계별로 어떻게 구현할지 제시합니다.

## 리포트 출력 형식

분석 완료 후 다음 형식으로 사용자에게 리포팅합니다:

```markdown
# 📋 프로젝트 맞춤형 기능 분석 리포트

## 🔍 프로젝트 성격 진단
- **프로젝트 유형**: [예: IDE 스타일 관리 도구, 데이터 대시보드 등]
- **핵심 유저 타겟**: [누가 이 툴을 쓸 것인가]

## 🚀 전략적 제안

### 🥇 최선순위: [기능명]
- 배경: ...
- 제안: ...
- 구현 전략: ...

### 🥈 차선순위: [기능명]
...

### 🥉 보완 제안: [기능명]
...

## 🛠 추천 기술/라이브러리
- [라이브러리명]: [추천 이유]

## ✅ 다음 단계 (Next Steps)
1. ...
2. ...
```

## 주의사항

- **투자 전용 아이디어에 한몰되지 않고**, 현재 프로젝트의 실제 코드 베이스와 목적(예: IDE 레이아웃, QT 관리 등)에 맞춘 실질적인 기능을 제안해야 합니다.
- **오버 엔지니어링**보다는 현재 구조에서 자연스럽게 확장 가능한 아이디어를 우선합니다.
- 사용자가 동의하면 `/feature-dev` 워크플로우로 즉시 전환하여 개발을 시작합니다.
