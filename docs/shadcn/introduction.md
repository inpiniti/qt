# Introduction

shadcn/ui는 아름답게 디자인된 접근 가능한 컴포넌트 세트이자 코드 배포 플랫폼입니다. 좋아하는 프레임워크 및 AI 모델과 함께 작동합니다. 오픈 소스. 오픈 코드.

## 컴포넌트 라이브러리가 아닙니다

**이것은 컴포넌트 라이브러리가 아닙니다. 이것은 컴포넌트 라이브러리를 구축하는 방법입니다.**

대부분의 전통적인 컴포넌트 라이브러리가 작동하는 방식은 다음과 같습니다:
- NPM에서 패키지를 설치
- 컴포넌트를 import
- 앱에서 사용

이 접근 방식은 디자인 시스템에 맞게 컴포넌트를 커스터마이징하거나 라이브러리에 포함되지 않은 컴포넌트가 필요할 때까지는 잘 작동합니다. 종종 라이브러리 컴포넌트를 래핑하거나, 스타일을 재정의하기 위한 해결 방법을 작성하거나, 호환되지 않는 API를 가진 다른 라이브러리의 컴포넌트를 혼합하게 됩니다.

## 핵심 원칙

shadcn/ui는 이러한 문제를 해결하기 위해 만들어졌습니다. 다음 원칙을 기반으로 구축되었습니다:

### 1. Open Code (오픈 코드)

컴포넌트 코드의 최상위 레이어가 수정을 위해 열려 있습니다.

- **완전한 투명성**: 각 컴포넌트가 어떻게 구축되었는지 정확히 볼 수 있습니다
- **쉬운 커스터마이징**: 디자인 및 기능 요구 사항에 맞게 컴포넌트의 모든 부분을 수정할 수 있습니다
- **AI 통합**: 코드에 대한 액세스로 LLM이 컴포넌트를 읽고, 이해하고, 심지어 개선하는 것이 간단해집니다

일반적인 라이브러리에서는 버튼의 동작을 변경해야 하는 경우 스타일을 재정의하거나 컴포넌트를 래핑해야 합니다. shadcn/ui를 사용하면 버튼 코드를 직접 편집하기만 하면 됩니다.

### 2. Composition (구성)

shadcn/ui의 모든 컴포넌트는 공통적이고 구성 가능한 인터페이스를 공유합니다.

- **예측 가능성**: 팀과 LLM 모두에게 예측 가능합니다
- **일관된 API**: 새로운 컴포넌트마다 다른 API를 배울 필요가 없습니다
- **서드파티 통합**: 서드파티 컴포넌트도 동일한 패턴을 따릅니다

컴포넌트가 존재하지 않으면 가져와서 구성 가능하게 만들고 나머지 디자인 시스템과 일치하도록 스타일을 조정합니다.

### 3. Distribution (배포)

shadcn/ui는 코드 배포 시스템이기도 합니다.

- **스키마**: 컴포넌트, 종속성 및 속성을 정의하는 플랫 파일 구조
- **CLI**: 크로스 프레임워크 지원으로 프로젝트 전체에 컴포넌트를 배포하고 설치하는 명령줄 도구

스키마를 사용하여 다른 프로젝트에 컴포넌트를 배포하거나 AI가 기존 스키마를 기반으로 완전히 새로운 컴포넌트를 생성하도록 할 수 있습니다.

### 4. Beautiful Defaults (아름다운 기본값)

shadcn/ui는 신중하게 선택된 기본 스타일을 가진 대규모 컴포넌트 컬렉션과 함께 제공됩니다.

- **즉시 사용 가능**: 추가 작업 없이 깔끔하고 미니멀한 모양을 갖춘 UI
- **통합 디자인**: 컴포넌트가 자연스럽게 서로 맞습니다. 각 컴포넌트는 다른 컴포넌트와 일치하도록 구축되어 UI의 일관성을 유지합니다
- **쉽게 커스터마이징 가능**: 무언가를 변경하려는 경우 기본값을 재정의하고 확장하는 것이 간단합니다

### 5. AI-Ready (AI 준비)

shadcn/ui의 디자인은 AI 도구가 코드와 함께 작동하기 쉽게 만듭니다.

- **오픈 코드**: AI 모델이 읽고 이해할 수 있습니다
- **일관된 API**: AI가 패턴을 학습하고 적용할 수 있습니다
- **자동 생성**: AI 모델이 컴포넌트 작동 방식을 학습하고 개선 사항을 제안하거나 기존 디자인과 통합되는 새로운 컴포넌트를 생성할 수 있습니다
